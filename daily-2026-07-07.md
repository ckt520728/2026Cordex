## 重要技術筆記（踩坑精華）

- **大型／vendored repo 在 Windows clone**：`Filename too long`（超過 260 字元 MAX_PATH）→ `git config --system core.longpaths true`（需管理員）或 clone 到短路徑；LFS smudge 失敗 → `GIT_LFS_SKIP_SMUDGE=1 git clone`
- **不要照抄 SKILL.md 的 API 範例**：`alphafold-database` 文件寫 v4 與 `confidenceScore`／`confidenceUrl`，但實際 API 已是 **v6**，confidence 欄位是 `plddtDocUrl` → 一定要先 `print(entry.keys())` 檢查真實 JSON
- **環境依賴**：`biopython` 未預裝，用 `Bio.*` 前先 `pip install biopython`；不確定時先用 stdlib（`urllib`、`json`）做 demo，不需安裝
- **臨床觀念坑**：基因在同一條染色體 ≠ 蛋白質相似。基因體位置相鄰多半是歷史偶然，不能推論蛋白質同源（例外：真正的複製 cluster 如 α-globin、HOX）
- **大型 repo 資安稽核法**：不要逐檔讀 5,000 檔，鎖定實際攻擊面 — manifest/`plugin.json` auto-run hook、`setup.py`/`package.json` install hook、CI workflow、pipe-to-shell（`curl|bash`）、base64/eval payload、reverse shell（`/dev/tcp`、`nc -e`）、credential-exfil 組合、raw-IP/webhook/C2、pickle 危險 opcode、hidden Unicode、agent-facing Markdown 的 injection 文字
- **pickle（.pkl）載入即執行程式碼**：用 `pickletools.genops` 掃 `os`/`subprocess`/`eval`/`socket` 的 GLOBAL/REDUCE opcode 再信任

## 關鍵結果（值得留存）

GRCh38 座標（Ensembl live）：

| Gene | Ensembl ID | chr16 position | Strand |
|---|---|---|---|
| HBZ | ENSG00000130656 | 142,728–154,503 | + |
| HBA2 | ENSG00000188536 | 172,021–173,910 | + |
| HBA1 | ENSG00000206172 | 176,660–177,527 | + |
| TSC2 | ENSG00000103197 | 2,047,345–2,089,491 | + |
| PKD1 | ENSG00000008710 | 2,088,708–2,135,898 | − |

- α-globin cluster 距 16p telomere 僅 ~172 kb → terminal deletion（ATR-16）最先波及它
- TSC2 與 PKD1 **尾對尾相鄰**（3′ 端重疊）→ 一個 deletion 造成 TSC2/PKD1 contiguous gene deletion syndrome
- α-globin ↔ PKD1 距離 = **1,914,798 bp（~1.91 Mb）**；同一 band（16p13.3），但位於兩端
- 臨床：ADPKD + α-thal trait 同時出現幾乎都是兩種常見疾病的巧合；只有大型（~1.9 Mb）16p13.3 deletion 才有真正關聯 → ADPKD 異常早發/嚴重且合併結節硬化症徵象或發展遲緩時，送 CMA 或 MLPA
