# Phase-one method routing

The source methods are adapted from the user's `data-analyzer`, `knowledge-base`, `research-organizer`, `academic-paper-deep-analysis`, and `paper-summary-extraction` skills.

## Preserve

- Computed numbers must come from executed code.
- Read complete source content before answering.
- Organize notes without confusing organization with validation.
- Analyze primary papers across concept, evidence, method, interpretation, critique, and implications.
- Mark abstract-only, OCR, and missing-supplement limitations.

## Override

- The project folder is the default output, not Obsidian.
- NotebookLM synthesis is a relational guide, not primary evidence.
- Use a common registry and claim ledger instead of skill-specific YAML dialects.
- Do not require unavailable PubMed, Notion, NotebookLM, or Obsidian tools; use local-file fallback.

## Source completeness levels

`full_primary`, `abstract_only`, `derived_note`, `metadata_only`, `unavailable`.
