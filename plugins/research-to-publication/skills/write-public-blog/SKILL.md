# Write Public Blog

Read `references/blog-contract.md` and use `assets/blog-outline.md`.

1. Confirm the public reader, target length, tone, and publication destination.
2. Invoke `$article-writer` to create and pause at an outline unless the user explicitly requested one-shot drafting.
3. Lead with the conclusion and one concrete problem.
4. Rebuild the argument for public comprehension; do not translate paragraph by paragraph.
5. Prefer natural Chinese. Add English in parentheses only at first use or when translation loses precision.
6. Keep paragraphs short and dependencies ordered.
7. Use examples to explain evidence boundaries, not to imply nonexistent validation.
8. Ensure every strong claim is no stronger than the approved ledger.
9. Include references, author identity, disclosure, and a non-diagnostic disclaimer when appropriate.
10. Stop for explicit content approval before invoking publication.

The evidence ledger and this skill override any generic article-writing suggestion that would introduce unsupported facts, fabricated first-person experience, or mandatory code blocks.
