## 技術筆記（踩坑與 learnings）

1. **週期 target 用 FORCE-Output 就夠，別用 FORCE-Internal**：三任務皆週期性，存在「完美的 W」，RLS 越訓越貼合；FORCE-Internal（訓 J）是為混沌 target 設計的（PITFALLS 失敗模式 6），對週期 target 是殺雞用牛刀。
2. **瀏覽器 60fps 的天花板來自 RLS 的 O(N²) P 矩陣更新**：Python 離線可用 N=1000，純 JS 要保 60fps 須收斂規模 → N 預設 220、上限 500，每幀 5 步、RLS 每 2 步。
3. **要永久可分享連結用 GitHub Pages，不要 Vercel MCP**：Vercel MCP 部署是 auth 保護站，分享連結 23h 過期；GitHub Pages 永久公開、免登入、免費，且 repo 已在 GitHub。`gh api -X POST repos/{owner}/{repo}/pages -f source[branch]=master -f source[path]=/` 即可啟用，首次建置約 1 分鐘。
4. **MCP 線上查詢不穩時，手邊跑過的本地 skill 程式碼是更可靠的 ground truth**：NotebookLM 查詢逾時就別死等，直接用已驗證的 `force_sinusoid.py` + `PITFALLS.md`。
5. **commit 時只 `git add` 自己的新檔**：claude-skills repo 工作目錄常有雜散未追蹤檔（`.claude/`、temp txt），別 `git add -A`；push 前常需 `git pull --rebase`。
